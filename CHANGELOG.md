# Changelog

## 0.3.0 - 2026-07-23

### Added

- 全面重写 Gradio 首页和对话工作区，加入品牌首屏、响应式布局、实时状态卡片、折叠式语音输入与更清晰的 Avatar 舞台。
- 新增桌面端与移动端自适应视觉系统，并将样式作为 Python 包资源发布。
- 新增悲伤、焦虑、恐惧等负面输入的表情保护测试，避免 Avatar 在安慰场景中错误微笑。
- 新增 1 个品牌图标与 5 张同风格品牌照片，并补充可直接交给设计 / 前端 Agent 的重设计说明。

### Changed

- Prometheus 音频播放改为按轮次保护的非阻塞完成任务：音频准备好后立即返回页面，表情和动作保持到语音结束后再复位。
- Live2D bridge 新增嵌入模式，主界面只展示角色舞台，调试信息仍保留在独立 bridge 页面。
- Live2D 角色按模型原始尺寸稳定缩放，并监听实际舞台容器尺寸变化，避免响应式布局切换后角色放大、偏移或被裁切。
- Live2D 语音增加页面可见性保护与浏览器级独占播放锁，避免多个已打开页面同时播放同一轮回复。
- 压力、焦虑、悲伤与恐惧场景默认使用 `concerned` 表情，减少不合时宜的 `soft_smile`。
- TTS 失败时不再播放 Mock 提示音冒充语音，而是保留文字回复并明确提示本轮语音不可用。

### Tests

- 新增 Prometheus 非阻塞播放、延迟复位、TTS 失败无提示音和负面情绪表情保护回归测试。
- 新增 Live2D 原始尺寸缩放与容器 ResizeObserver 回归断言，并完成 1440px / 390px 浏览器视觉检查。

## 0.2.0 - 2026-07-22

### Fixed

- 按 Gradio Session ID 隔离对话历史和动作冷却，并让清空按钮同步清理后端状态。
- 串行化 Avatar 播放，避免并发请求造成表情、动作和音频交错。
- 为不完整或越界的 LLM 控制字段提供默认值与范围校验，避免回显无效 JSON。
- 将语速传给 TTS，将动作强度传给 Avatar；限制过长的说话前停顿。
- 修正 Edge TTS 音频格式：通过 ffmpeg 将 MP3 转为真正的 WAV，并加入超时处理。
- 停用根据系统自己选出的动作自动强化用户偏好的错误反馈回路。
- 对用户画像文件名进行安全处理，防止 Session ID 越界写入画像目录。
- 修复本地评测脚本对会话化历史接口的调用。
- 清空 Gradio 会话时同步移除 Live2D bridge 中的上一轮文本、音频和事件。

### Changed

- Prometheus bridge 使用轮次状态、原子状态文件、可访问音频 URL 和 250 ms 轮询。
- Live2D 浏览器端播放生成音频、同步口型，并按动作强度调整参数幅度。
- Gradio 在 Prometheus 模式下自动启动并嵌入本地 bridge，默认仅监听 `127.0.0.1`。
- `scripts/run_gradio.py` 完全尊重 `.env` 中的 provider 与网络配置，不再强制覆盖。
- 真实 API Prometheus 冒烟脚本不再强制使用本地 MOSS TTS，并尊重 `.env` 的 key 变量与 TTS 配置。
- OpenAI SDK 内部重试关闭，由管线统一执行重试策略。

### Tests

- 新增会话隔离、清空、动作冷却、显式偏好反馈、字段兼容、参数校验和 Prometheus 状态回归测试。
