# HAI 情感型 AI 虚拟陪伴 — 前端设计规范（DESIGN.md）

> 本文件是从零开发 HAI 独立响应式前端的**唯一风格依据**。所有数值均可直接落地。
> 语境：从 Gradio 页面迁出的独立前端，不绑定任何框架；必须尊重现有 `respond` / `clear` / ASR / 音频 / Live2D bridge 能力。当前接口为**整轮完成后返回**，严禁伪装成 token streaming。

---

## Overview

HAI 是一个"我说一句话，角色理解我的情绪，用合适的内容、语气、表情和动作回应我"的夜间陪伴产品。用户首先应该看到一个**正在倾听的角色**，而不是聊天窗口、参数面板或营销拼贴。

**它像什么**：深夜书桌上一盏小台灯——深海军蓝的安静夜色里，一圈温暖的琥珀色光，一个会看你、会点头、会回应的二维卡通角色。
**它不像什么**：不像工程控制台（高密度状态卡片、等宽字段铺满），不像普通客服聊天机器人（头像+气泡列表居中堆叠），不像营销落地页（hero + 功能卡片 + 图库同权拼接），也不像"紫蓝渐变 + 玻璃拟态卡片"的 AI 模板页。

**本 case 的大胆设计主张：「平面夜色 + 台灯暖光」**
不用渐变、不用玻璃、不用发光描边。用一整块**平面深海军蓝**作为夜的底色，用**克制的琥珀暖色**作为唯一的光源隐喻：CTA、聚焦态、正在说话的角色状态环，都取台灯暖光的逻辑——**暖色是稀缺的，只在"角色正在回应你"和"你可以行动"的地方出现**。青色（月牙发夹色）只做冷静的次级点缀，珊瑚色只给人情味时刻（录音、情绪反馈、局部警告）。配合柔和赛璐璐风格的原创角色插画与轻微纸张纹理，营造"手绘的夜晚"，而不是"渲染的未来"。

**识别度来源**：① Live2D 角色与对话、输入共处的**单一舞台**（不是聊天列表）；② 琥珀暖光 vs 平面夜色的强对比用色纪律；③ 月牙青作为贯穿图标、徽章、链接的签名色；④ 首次进入的"引导氛围态"与对话中的"沉浸会话态"两种明确的界面形态切换。

## Colors

背景基调为深色（产品气质决定：夜间陪伴），但**禁止大面积渐变、禁止紫蓝霓虹、禁止玻璃拟态**。所有背景是平面色 + 极轻纸张噪点纹理（opacity ≤ 4% 的 tile，可用 SVG feTurbulence 生成）。

### 深色主题（默认且唯一主题；不做亮色切换）

| Token | HEX | 用途 |
|---|---|---|
| `night-0` | `#0E1626` | 页面最底层背景（平面色，非渐变） |
| `night-1` | `#14203A` | 舞台容器、对话区背景 |
| `night-2` | `#1B2A4A` | 卡片 surface、输入框、折叠面板 |
| `night-3` | `#24365C` | hover 态 surface、分隔层级 |
| `line` | `#2E4170` | 1px 边框、分隔线（不用纯白低透明度模拟） |
| `ink-strong` | `#F2EEE6` | 主文字（暖白，不用纯蓝白） |
| `ink` | `#C9CEDD` | 正文、对话气泡文字 |
| `ink-dim` | `#8A93AC` | caption、时间戳、placeholder（仅非关键信息） |
| `amber` | `#F2A93B` | **主强调色**：主按钮、focus ring、角色"说话中"状态环、发送键 |
| `amber-deep` | `#D98F22` | 主按钮 hover/pressed |
| `amber-soft` | `#3A2F1A` | 琥珀色的深色底（选中底色、tag 底） |
| `moon` | `#5FD4D0` | 签名青（月牙发夹色）：链接、次级图标、badge、"倾听中"呼吸点 |
| `moon-dim` | `#3B8A8C` | 青的 hover/深态 |
| `coral` | `#E8846B` | 人情色：录音中、情绪反馈标记、局部警告、温暖提示 |
| `ok` | `#7BC47F` | 成功/已连接（克制使用） |
| `warn` | `#E8B84B` | 降级警告（TTS 失败但文字成功等） |
| `danger` | `#E0604F` | 失败、清空确认按钮 |

### 用色纪律

- **60/30/10**：`night-0/1` 系 ≈ 60%，`night-2/3` surface ≈ 30%，琥珀+青+珊瑚合计 ≤ 10%。
- **琥珀是稀缺的**：一屏内琥珀色实心元素不超过 2 处（通常是发送按钮 + 一个状态指示）。禁止用琥珀做大面积背景。
- **珊瑚只给"人和情绪"**：录音波纹、情绪 tag、局部警告。不给系统性 UI（tab、链接）。
- **禁止**：紫罗兰/电蓝渐变、`rgba(255,255,255,0.06)` 式玻璃卡片、发光描边（glow box-shadow）、彩色左边框卡片、霓虹色。
- 文字对比：`ink-strong` on `night-1` ≈ 13:1；`ink` on `night-1` ≈ 8:1；`ink-dim` 不用于关键状态文字（低于 4.5:1 的内容只允许是装饰性）。

## Typography

中文为主产品，字体选择以中文为先；西文/数字为搭配。全部可免费加载（Google Fonts）。

| 角色 | 字体 | 字号 / 字重 / 行高 |
|---|---|---|
| 页面大标题（首入 hero） | 中文 `Noto Serif SC` 700，西文 `Sora` 600 | 40px / 1.25 / letter-spacing 0.01em |
| Section 标题 | `Noto Serif SC` 600 | 24px / 1.35 |
| 角色名 / 舞台标题 | `Noto Serif SC` 600 | 18px / 1.4 |
| 正文 / 对话气泡 | `Noto Sans SC` 400（西文 fallback `Sora` 400） | 16px / 1.75 |
| 按钮 | `Noto Sans SC` 500 | 15px / 1.2 / letter-spacing 0.02em |
| 输入框 | `Noto Sans SC` 400 | 16px / 1.6 |
| Caption / 时间戳 / placeholder | `Noto Sans SC` 400 | 13px / 1.5 |
| 诊断字段 / 延迟 / 枚举值 | `IBM Plex Mono` 400 | 12.5px / 1.6 |

**字体性格**：`Noto Serif SC` 的衬线给标题一点"夜里读书"的纸感与温度，避免无衬线全场的冷冰冰工具感；`Sora` 是几何但圆润的 grotesque，做拉丁搭配而不抢戏；`IBM Plex Mono` **只允许出现在诊断区**（emotion/latency 等枚举字段），用位置而非字号来建立"这是给开发者看的"的语义。

**规则**：
- 对话气泡内文字最大宽度 56ch，长中文自然换行，不断词。
- 标题不大写、不拉伸字距；大标题只在首次进入态出现一次，对话态不再有 display 级标题。
- 数字（延迟、计时）统一用 `IBM Plex Mono` 的 tabular figures。

## Layout

**栅格**：12 列，gutter 24px，页面最大内容宽度 1280px，页面左右 padding：桌面 48px / 平板 32px / 移动 20px。垂直节奏以 8px 为基础单位，section 间距 96px（桌面）/ 64px（移动）。

### 两种界面形态（核心布局规则）

**形态 A · 首次进入 / 空对话（引导态）**
- 非对称 70/30 构图：左 7 列为文字区（衬线大标题 40px + 一句 16px 引导文案 + 主 CTA"开始聊一聊" + 最多 2 个示例问题 chip），右 5 列为角色舞台（Live2D 已加载并 idle，叠加一句"嗨，我在。"的引入气泡）。
- 首屏只有这一个视觉主张。下方（首屏之外）才是产品叙事区，最多 4 个叙事 slot（倾听 / 自然语音 / 表情动作 / 安静陪伴），交错双栏图文，**不是四张同权卡片**。
- 不出现图库墙、不出现"功能 × 6"图标网格。

**形态 B · 对话进行中（沉浸态）——这是默认主舞台**
- 进入对话后：营销内容、大标题、示例 chip **全部退出主舞台**（不再占据会话区空间，叙事区仅在页脚上方保留可滚动到达的一节）。
- 一个**统一的舞台容器**（`night-1`，圆角 20px，1px `line` 边框）承载三要素：
  - 左 5 列：Live2D 角色舞台（见 Live2D 硬约束）。
  - 右 7 列：对话历史（纵向滚动区）+ 底部固定输入栏。对话历史与输入栏同属一个 flex 列，输入栏 `position: sticky` 于容器底部，padding 16px，上缘 1px `line` 分隔。
- 角色回复**只出现一次**：文字在对话气泡里，角色负责表演；舞台内不再重复显示同一段大文本。
- 技术诊断（emotion / expression / gestures / voice / latency / warnings）为对话区下方一条**默认折叠**的通栏"本轮表现"抽屉（见 Components → Diagnostics drawer）。

### Live2D 舞台硬约束（不可违反）

- 缩放采用 `contain` 思路，**按模型原始尺寸**计算缩放比，禁止根据已缩放的宽高反复换算。
- 四周安全空间 ≥ 12%；头顶、双手、身体下沿不被裁切。
- 桌面：角色舞台占舞台容器宽度 40%–46%，高度 560–680px，角色视觉中心**高于**容器几何中心约 6%。
- 窄屏（< 768px）：角色舞台高度 34–42vh，最小 320px、最大 480px；优先保证角色完整，允许舞台容器两侧留白，不允许裁切角色。
- 角色 idle 动效轻微（±2px 呼吸浮动，4s 周期）；容器本身**不**晃动、不漂浮。
- 加载中：`moon` 色细线圆环 spinner + "正在唤醒角色…"；失败：`coral` 小图标 + 一句说明 + "重试"按钮；bridge 断开：舞台顶部一条 `coral` 细横幅（不弹 modal）。绝不显示空白 iframe。

### 内容密度

对话气泡间距 16px；气泡 padding 14px 18px；气泡最大宽度 = 对话列宽 × 78%。用户气泡右对齐、角色气泡左对齐，同侧连续两条气泡间距缩为 6px。

## Elevation & Depth

层级靠**平面色明度差 + 1px 边框 + 极克制的暖色阴影**表达，**不用玻璃拟态、不用大模糊投影、不用发光描边**。

- 层级阶梯：`night-0`（页面）→ `night-1`（舞台）→ `night-2`（卡片/输入框）→ `night-3`（hover）。相邻层明度差即层级。
- 卡片/浮层边框统一 `1px solid line`。
- 阴影仅两处合法使用：
  - 主 CTA / 发送按钮：`0 4px 16px rgba(242,169,59,0.22)`（台灯光晕感，唯一带色相的阴影）。
  - Modal / dropdown / 清空确认浮层：`0 12px 32px rgba(4,8,18,0.55)` + `1px solid line`。
- Sticky 元素（输入栏、顶栏）：背景用**不透明** `night-1`（顶栏用 `night-0`），不加 backdrop-filter，不加磨砂。
- 角色"表演中"状态：角色舞台外缘 2px 琥珀描边以 2.4s 周期在 40%–100% 透明度间呼吸——这是唯一的"光效"，且不是 box-shadow glow。

## Shapes

形状语言：**安静的圆角**——柔和但不幼稚，介于硬朗控制台与儿童圆角之间。

| 元素 | 圆角 |
|---|---|
| 舞台容器 / 叙事大图 | 20px |
| 对话气泡 | 16px（角色气泡左上角 6px，用户气泡右上角 6px，形成朝向感） |
| 卡片 / 输入框 / 折叠面板 | 12px |
| 主按钮 / 次按钮 | 999px（pill）——全站唯一 pill 形状，让"行动"一眼可辨 |
| Badge / tag / chip | 999px |
| 录音按钮 / 头像 / icon button | 正圆 |
| Modal / dropdown | 16px |

- 插画一律以 `object-fit: cover` 置入圆角容器，容器圆角 20px。
- 不出现斜切、六边形、波浪分隔线等异形。

## Components

### Top navigation（高 64px，sticky，`night-0` 不透明背景，下缘 1px `line`）
- 左：简化头像 logo（32px 正圆）+ "HAI" 字样（`Noto Serif SC` 600，18px）+ 一句 13px `ink-dim` 副标"安静的 AI 陪伴"。
- 右：连接状态点（8px 圆，`ok` 已连接 / `coral` bridge 离线，配 13px 文字）→ 设置 icon button（40px 正圆，`night-2` 底，hover `night-3`）。
- 提供商名称、Mock/真实模式说明**只**出现在设置面板和诊断区，顶栏不放徽章。
- 移动端副标隐藏，只留 logo + 状态点 + 设置。

### Hero（仅形态 A 出现）
- 布局见 Layout。无背景大图铺底；氛围靠平面夜色 + 右侧角色 + 一句琥珀色点缀词（标题中最多一个词用 `amber`）。
- 主 CTA"开始聊一聊"为琥珀实心 pill（见 Primary button）；下方最多 2 个示例问题 chip（`night-2` 底，`moon` 色文字，pill，hover `night-3`）。

### Primary button（发送 / 开始聊一聊 / 重试主操作）
- 背景 `amber`，文字 `#1A1206`（深棕黑，保证对比），`Noto Sans SC` 500 15px，padding 12px 28px，pill，高度 44px。
- Hover：背景 `amber-deep`，阴影加深。Active：translateY(1px)。Disabled：背景 `night-3`，文字 `ink-dim`，无阴影，cursor not-allowed。
- 禁止：琥珀底配白字（对比不足）、ghost 风格琥珀描边按钮（琥珀描边只留给角色状态）。

### Secondary button（清空对话、查看诊断等）
- 背景透明，`1px solid line`，文字 `ink`，pill，同尺寸。Hover：背景 `night-2`。危险动作（清空确认）文字与边框改用 `danger`。

### Chat bubble
- 角色气泡：背景 `night-2`，文字 `ink-strong`，16px/1.75，左对齐，左上角小圆角。气泡上方一行 12.5px meta：`moon` 色角色名 + `ink-dim` 时间戳；若本轮有情绪标签，追加一个情绪 tag（见 Badge）。
- 用户气泡：背景 `#2A3B63`（比 night-3 深半阶的蓝），文字 `ink-strong`，右对齐。
- 表演中：角色气泡左缘有 3px 宽、8px 高的琥珀小圆点在气泡头部闪烁（音频播放期间）。
- 语音可用时，角色气泡尾部有一个 28px 正圆"重播"icon button（`moon` 图标，透明底，hover `night-3`）。
- 语音失败：气泡尾部显示 `warn` 色 13px 文字"语音不可用" + "重试语音"链接（`moon` 色），**保留全部文字内容**，不播放占位音。
- 失败的用户轮次：用户气泡下方出现 `danger` 13px "发送失败，点击重试"，**输入内容保留在输入框或失败气泡内**，不清空。

### 等待 / 进度指示（替代"三个点打字中"）
当前接口整轮返回，用**阶段式进度条**表达"理解 → 回复 → 声音 → 表演"，不伪装逐字输出：
- 角色气泡位置先出现一个 `night-2` 气泡骨架：内部为一条横向 4 段步骤指示（4 个 6px 圆点 + 12.5px 阶段名，`ink-dim`），当前阶段圆点为 `amber` 并以 1.2s 呼吸；已完成阶段变 `moon`。
- 文字到达后骨架替换为文字气泡；TTS 阶段继续由气泡尾部的 13px "正在生成语音…"（`ink-dim`）表达，最长 15s 超时后转为"语音不可用 + 重试"。
- 整体请求超过 30s：气泡内追加 `warn` 提示"这次想得有点久，仍在处理中"，不打断等待。

### Input bar（对话列底部 sticky）
- 容器：`night-2`，圆角 12px，1px `line`；聚焦时边框变 `amber` + 外缘 2px `rgba(242,169,59,0.25)` 光环（focus ring 语义，非装饰 glow）。
- 结构：左起 [录音 icon button 40px 正圆] [多行输入，1–5 行自适应，超过后内部滚动] [发送 Primary button，图标+文字"发送"]。
- 空输入时发送按钮 disabled；Enter 发送、Shift+Enter 换行；placeholder 用"说点什么，我在听。"（`ink-dim`）。
- **录音中**：录音按钮变 `coral` 实心 + 1.6s 扩散环动画；输入框区域显示 13px 计时（`IBM Plex Mono`）与"停止"次按钮；波形用 5 根 `coral` 竖条（高 4–16px 随机律动）。
- **转写中**：录音按钮位置显示 spinner + "转写中…"；**转写失败**：`danger` 提示"没听清，请再试一次"，麦克风按钮恢复原状，不丢失之前已输入的文字。

### Card（叙事区专用，会话区不用卡片拼贴）
- `night-2` 底，圆角 12px，1px `line`，padding 24px。无左边框色条、无图标网格、无 hover 浮起。hover 仅背景升 `night-3`。
- 叙事区用"图 + 文"交错双栏，不用三列卡片墙。

### Badge / Tag
- 情绪 tag（支撑 supportive、开心 happy 等）：`amber-soft` 底、`amber` 文字，12.5px，pill，padding 4px 12px。仅出现在角色气泡 meta 行与诊断区。
- 状态 badge（演示数据 / 真实服务）：`night-2` 底、`moon` 文字、1px `moon-dim` 描边，仅诊断区与设置面板出现。
- 警告 badge：`coral` 文字 + 透明底，配 14px 警告图标。

### Tab / Filter（诊断区内：本轮表现 | 会话信息）
- 下划线式：文字 14px `ink-dim`，选中 `ink-strong` + 底部 2px `amber` 短下划线（长度 = 文字宽，非通栏）。不用胶囊 tab 墙。

### Modal / Dropdown
- Modal（清空确认）：居中 400px 宽，`night-1` 底，圆角 16px，`0 12px 32px rgba(4,8,18,0.55)`，遮罩 `rgba(6,10,20,0.7)` 不透明遮罩（不模糊）。主按钮"清空"为 `danger` 实心，次按钮"取消"。确认后对话区回到空对话态（形态 B 内的空态，不跳回营销 hero）。
- Dropdown（设置面板项）：`night-2` 底，1px `line`，圆角 12px，选项 hover `night-3`，选中项左侧 3px `amber` 竖条。

### Diagnostics drawer（"本轮表现"）
- 对话区下方通栏，默认折叠为一条 44px 高横条：左侧 `IBM Plex Mono` 12.5px "emotion · expression · gestures · latency" 摘要，`ink-dim`；右侧"展开"chevron。
- 展开后：`night-2` 面板内以 2 列（移动端 1 列）键值对展示全部字段与分阶段延迟；warnings 逐条以 `warn` 色列出。mono 字体仅在此区使用。

### Footer
- `night-0`，上缘 1px `line`，padding 32px。左侧一句 13px `ink-dim` 项目说明，右侧"技术诊断入口 / 设置"文字链接（`moon`）。无多列 sitemap、无社交图标墙。

## Imagery / Media

**当前阶段没有已批准的位图资产**——界面在所有图片槽位为空时必须完整可用（无破图、无空缺白块，槽位整体隐藏并回收布局空间）。`assets/brand/` 下旧摄影图**禁止使用**。

### 新媒体方向：原创二维卡通 / 柔和赛璐璐
同一位原创、非 IP、中性气质的年轻陪伴角色：**午夜蓝短发、青色月牙发夹、琥珀色眼睛、珊瑚色内领**。柔和赛璐璐上色、干净轮廓、少量手绘纸张纹理、简单安静的夜间小场景（如台灯、窗台、月色），**无复杂背景、无文字水印**。

### 槽位、比例与裁切规则

| 槽位 | 比例 | 用途与规则 |
|---|---|---|
| Logo / 头像 | 1:1，32/40px 正圆裁切 | 简化头像（头部特写，月牙发夹可见）；favicon、顶栏、角色气泡 meta 左侧 |
| 空对话引入 | 3:4 竖版，圆角 20px | 仅当 Live2D 不可用时作为角色舞台**降级**视觉；正常态主视觉是 Live2D，不重复放插画 |
| 叙事插图 ×4 | 4:3 横版，圆角 20px | 倾听 / 自然语音 / 表情动作 / 安静陪伴；交错双栏图文，每屏最多 1 张 |
| 页脚上方横幅 | 21:9（可选） | 最末叙事段，可无 |

- **插画永不压过 Live2D**：会话态舞台内不放任何插画；叙事插图最大宽度 560px。
- 裁切：`object-fit: cover`，角色头部与月牙发夹必须在安全区内（上下各留 ≥ 10%），不裁掉发夹与表情。
- 降级：槽位无图时，该图文块整体隐藏（不显示占位灰框）；角色舞台降级用 `night-2` 底 + 月牙线稿 SVG（`moon` 色，32px）+ "角色暂时不在线" + 重试按钮。
- **禁止**：照片写实、3D 渲染、塑料质感、儿童低幼风、日系 IP 模仿、文字压图（标题不覆盖在插画上）、全 bleed 大图背景。

## Motion

**克制的生命感**：动效服务于"角色是活的"，UI 本身安静。全部动画尊重 `prefers-reduced-motion`（开启后：呼吸/浮动/波纹全部静止，仅保留 opacity 渐变）。

| 场景 | 参数 |
|---|---|
| 通用 hover / 颜色过渡 | 150ms ease-out |
| 气泡进入 | 200ms ease-out，translateY 8px + opacity 0→1 |
| 抽屉展开（诊断/设置） | 250ms cubic-bezier(0.4, 0, 0.2, 1)，高度动画 |
| 角色 idle 呼吸 | ±2px，4s ease-in-out 循环 |
| 倾听中状态点（moon） | 1.8s 透明度呼吸 |
| 说话中琥珀描边 | 2.4s 透明度 40%↔100% |
| 录音扩散环 | 1.6s ease-out 循环，scale 1→1.5 + fade |
| 阶段进度点 | 当前阶段 1.2s 呼吸 |
| Modal 进入 | 200ms，scale 0.96→1 + fade |
| 形态切换（首入 → 沉浸） | 400ms：营销区 opacity/高度收起，舞台容器展开，**一次性**，随后不再跳动 |

- Carousel/滚动揭示：叙事区滚动进入时仅 12px translateY + fade（200ms），逐块错开 60ms；不做 parallax、不做横向滚动画廊。
- **不动**：容器整体晃动、按钮持续脉冲、背景粒子/星空动画、打字机式逐字假装流式、任何循环的发光。

## Do's and Don'ts

**必须做**
- Live2D 角色 + 对话历史 + 输入栏共处于**一个舞台容器**；角色回复文字只出现一次。
- 首次进入（引导态）与对话中（沉浸态）是两种明确形态；开始后营销内容退出主舞台。
- 琥珀色稀缺使用：主 CTA、focus、"角色正在回应"状态，一屏 ≤ 2 处实心琥珀。
- 等待用"理解 → 回复 → 声音 → 表演"四阶段进度；失败保留用户输入并提供局部重试。
- 技术诊断默认折叠为一条横条；mono 字体只出现在诊断区。
- 无图状态下界面完整成立；角色舞台加载/失败/bridge 断开均有克制占位与重试。
- 支持 `prefers-reduced-motion`；移动端输入栏贴视口底部且不被软键盘遮挡。

**禁止做**
- 禁止深色 + 紫蓝渐变、玻璃拟态卡片、发光描边、彩色左边框卡片、图标功能网格、高密度等宽字体铺满主界面。
- 禁止"暖米底 + 衬线大标题 + 杂志留白"的编辑风模板（衬线在此只做深色夜底上的标题，不做米色纸面）。
- 禁止普通聊天机器人布局：全宽居中气泡列表、角色只是 24px 小头像。
- 禁止营销卡片拼贴：hero + 6 功能卡 + 图库同权堆叠。
- 禁止伪装流式：打字机逐字、假进度百分比。
- 禁止 TTS 失败时播放占位音；禁止错误清空用户输入。
- 禁止复用 `assets/brand/` 旧摄影图；禁止照片、3D、IP 模仿插画。
- 禁止文字压图、full-bleed 背景图、星空粒子背景。
- 禁止把提供商/模型名做成顶栏徽章。

## Responsive Behavior

| 断点 | 布局 |
|---|---|
| ≥ 1200px | 12 列；舞台容器内角色 5 列 + 对话 7 列；舞台高 560–680px；输入栏 sticky 于对话列底部 |
| 768–1199px | 角色舞台在上（高 420–520px，contain 完整角色）+ 对话在下，同一容器；或角色收窄为 4 列的紧凑双栏（内容宽度 ≥ 960px 时优先紧凑双栏）。无横向滚动 |
| < 768px（含 390px） | 单列：顶栏 → 角色舞台（34–42vh，min 320px / max 480px）→ 对话历史（获得剩余主要纵向空间）→ 输入栏 sticky 视口底部。诊断抽屉、设置、叙事区全部单列 |

- 390px 下：气泡最大宽度 = 屏幕宽 − 88px；气泡 padding 缩为 12px 14px；叙事图文块的垂直间距从 96px 缩到 64px；叙事插图保持 4:3，宽度撑满列。
- 输入栏在软键盘弹出时保持在可视区底部（使用 `100dvh` 与 `visualViewport` 补偿）；录音按钮与发送按钮不小于 44px。
- 移动端 hero（引导态）：文字区在上、角色在下，标题缩至 28px；示例 chip 允许横向滑动换行。
- 任何断点下角色不被裁切、无横向滚动。

## Accessibility

- 点击目标 ≥ 44×44px（icon button 视觉 40px 时用透明热区补足）。
- 对比度：正文 ≥ 4.5:1，大标题 ≥ 3:1；`ink-dim` 不用于关键状态；琥珀按钮文字用 `#1A1206` 不用白色。
- Focus ring：2px `amber` 外环 + 2px 偏移，所有可交互元素可见；不用 `outline: none` 无替代方案。
- Icon button（录音、重播、设置、清空）必须有 `aria-label`；录音中为"停止录音"，重播为"重播这条语音"。
- 表单错误：输入区错误用 `danger` 13px 文字 + 图标内联提示，并 `aria-live="polite"`；转写失败、发送失败均通过 `aria-live` 区域播报。
- 阶段进度与"语音生成中"等状态变化对屏幕阅读器以文字等价呈现（"正在生成语音"），不只靠动画。
- 键盘：Enter 发送 / Shift+Enter 换行；Esc 关闭 modal 与设置面板；Tab 顺序为 顶栏 → 角色舞台（可聚焦重试）→ 对话历史 → 输入栏 → 诊断抽屉；modal 打开时焦点圈定在 modal 内。
- `prefers-reduced-motion`：所有呼吸、浮动、波纹动画静止；阶段进度改为静态高亮当前步骤文字。
- 录音依赖麦克风权限：权限被拒时给出文字说明与文字输入引导，不只红点亮灭。
